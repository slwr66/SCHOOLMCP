# План разработки и изучения MCP серверов (Images & Quiz)

## 🎯 Обзор целей
За 3 дня (10-12 часов) вы создадите два полноценных MCP сервера:
1.  **Images-MCP:** Поиск изображений через Unsplash API с соблюдением правил атрибуции.
2.  **Quiz-MCP:** Генератор викторин на базе OpenTDB с автоматическим переводом на русский язык (Yandex Translate) и экспортом результатов.

---

## 📅 День 1: Основы FastMCP и интеграция Unsplash (4 часа)

**Цель:** Настроить окружение, изучить архитектуру FastMCP и реализовать поиск изображений.

### 1. Теория и Настройка (1 час)
**Задачи:**
1.  **Изучение FastMCP:**
    *   Понять концепцию: Инструменты (Tools) vs Ресурсы (Resources).
    *   Архитектура "Single Instance": Почему `mcp = FastMCP()` должен быть в отдельном файле.
    *   Асинхронность: Почему `async def` и `httpx` обязательны для I/O операций.
    *   *Ресурс:* [Документация FastMCP](https://gofastmcp.com)
2.  **Unsplash API:**
    *   Регистрация на [Unsplash Developers](https://unsplash.com/developers).
    *   Получение `Access Key` (Demo mode: 50 запросов/час).
    *   **Важно:** Изучение правил атрибуции (обязательное указание автора и UTM-меток) и Hotlinking (использование прямых ссылок).
3.  **Инициализация проекта:**
    *   Установка `uv` (менеджер пакетов).
    *   Создание структуры папок.

**Команды установки:**
```bash
mkdir images-mcp && cd images-mcp
uv init
uv add fastmcp httpx python-dotenv
```

### 2. Практика: Кодинг Images-MCP (2.5 часа)
Реализуйте сервер, следуя строгому разделению ответственности.

#### Структура проекта
```text
images-mcp/
├── .env                 # UNSPLASH_ACCESS_KEY=...
├── mcp_instance.py      # Единая точка входа
├── server.py            # Запуск
└── tools/
    ├── __init__.py
    └── get_images.py    # Логика инструмента
```

#### Шаг A: Единый инстанс (`mcp_instance.py`)
```python
from fastmcp import FastMCP
mcp = FastMCP("Images MCP", dependencies=["httpx", "python-dotenv"])
```

#### Шаг B: Инструмент поиска (`tools/get_images.py`)
*Ключевые требования:*
*   Используйте `httpx.AsyncClient`.
*   Обработайте параметры: `topic`, `prompts` (список), `count` (1-10).
*   В ответе сформируйте поле `attribution` согласно гайдлайнам Unsplash.

```python
# Фрагмент логики атрибуции
author_name = photo['user']['name']
author_link = f"{photo['user']['links']['html']}?utm_source=images-mcp&utm_medium=referral"
attribution = f"Photo by {author_name} on Unsplash"
```

#### Шаг C: Точка входа (`server.py`)
```python
from mcp_instance import mcp
from tools.get_images import get_images 
# Импорт обязателен для регистрации декораторов!

if __name__ == "__main__":
    mcp.run()
```

### 3. Тестирование и Отладка (0.5 часа)
*   Запустите инспектор: `fastmcp inspect server.py`
*   Проверьте запрос: `topic="cat"`, `count=1`.
*   **Советы по отладке:**
    *   *Ошибка 403:* Скорее всего, закончился лимит 50 запросов/час или неверный ключ.
    *   *Пустой результат:* Проверьте, что `httpx` не выдает исключений (оберните в try/except).

**✅ Чеклист Дня 1:**
- [ ] Сервер запускается через `uv run server.py`.
- [ ] Инструмент возвращает URL картинки и строку атрибуции.
- [ ] API ключ скрыт в `.env`.
- [ ] Используется асинхронный `httpx`.

## 📅 День 2: Сложная логика и цепочки API (4 часа)

**Цель:** Создать Quiz-MCP, интегрировать OpenTDB и Yandex Translate, реализовать пакетный перевод.

### 1. Изучение API и Подготовка (1 час)
**Задачи:**
1.  **OpenTDB API:**
    *   Эндпоинт: `https://opentdb.com/api.php`.
    *   Лимит: 1 запрос с одного IP раз в 5 секунд (нужно учесть в коде!).
    *   Особенность: Текст приходит с HTML-сущностями (`&quot;`), нужен `html.unescape`.
2.  **Yandex Translate API:**
    *   Получение `IAM_TOKEN` или `API_KEY` в Yandex Cloud.
    *   Изучение метода **Batch Translation** (перевод массива строк за один запрос). Это критично для производительности.

### 2. Практика: Кодинг Quiz-MCP (2.5 часа)

#### Структура проекта
Аналогична Day 1, но добавляем файл утилит.
```text
quiz-mcp/
├── tools/
    ├── get_quiz.py
    └── utils.py      # Клиенты OpenTDB и Yandex
```

#### Шаг A: Утилиты (`tools/utils.py`)
Реализуйте две ключевые функции:
1.  `fetch_category_id(topic)`: Поиск ID категории в OpenTDB по названию.
2.  `translate_batch(texts, target="ru")`: Отправка списка строк в Yandex.
    *   *Совет:* Yandex принимает JSON `{ "texts": ["str1", "str2"], ... }`.

#### Шаг B: Основная логика (`tools/get_quiz.py`)
Алгоритм инструмента:
1.  Найти категорию по теме (fallback на "General Knowledge" если не найдена).
2.  Запросить вопросы у OpenTDB.
3.  **Оптимизация:** Собрать все тексты (вопросы + правильные ответы + неправильные ответы) в один плоский список.
4.  Отправить плоский список в `translate_batch`.
5.  Разобрать переведенный список обратно в структуру вопросов.

```python
# Пример "выпрямления" списка для перевода
text_pool = []
for q in questions:
    text_pool.append(q['question'])
    text_pool.append(q['correct_answer'])
    text_pool.extend(q['incorrect_answers'])
# ... вызов перевода ...
# ... обратная сборка ...
```

### 3. Отладка и Проверка (0.5 часа)
*   Протестируйте через Инспектор.
*   **Проблемы с Rate Limits:** Если OpenTDB возвращает ошибку, добавьте `await asyncio.sleep(5)` перед повторной попыткой или кеширование.
*   **Кодировка:** Убедитесь, что кавычки в вопросах отображаются корректно (не `&quot;`).

**✅ Чеклист Дня 2:**
- [ ] Викторина генерируется на русском языке.
- [ ] Используется пакетный перевод (1 запрос к Yandex на всю викторину).
- [ ] Ошибки OpenTDB (например, "нет вопросов в категории") обрабатываются корректно.

### ⚠️ Типичные ошибки и решения
1.  **Ошибка:** `RuntimeError: This event loop is already running`.
    *   *Решение:* Используйте `fastmcp inspect` или `asyncio.run()` только в точке входа тестов.
2.  **Ошибка:** `httpx.ConnectError`.
    *   *Решение:* Проверьте VPN (для Yandex/OpenTDB из некоторых регионов) или настройки прокси.
3.  **Проблема:** Unsplash Rate Limit (403).
    *   *Решение:* В коде `get_images` добавьте обработку кода 403 и возвращайте понятное сообщение пользователю ("Лимит демо-режима исчерпан").
